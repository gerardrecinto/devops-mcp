# Go-To-Market

## Positioning

DevOps MCP is a read-only operational control plane for AI assistants. It lets engineers ask plain-English questions across Kubernetes, Jenkins, GitHub Actions, cloud logs, ServiceNow, Slack, and Teams.

## Best First Customer

Small to mid-size platform teams that already use Claude Desktop or internal LLM tools and lose time switching across dashboards during incidents.

## Monetization Path

| Tier | Offer | Buyer |
|---|---|---|
| Open Source | Local read-only MCP server with core integrations. | Individual engineers. |
| Team | Shared config templates, audit retention, packaged deployment, support. | DevOps/SRE teams. |
| Enterprise | SSO, policy controls, approval workflows, private integrations, managed rollout. | Enterprise platform orgs. |
| Services | Integration setup, runbook grounding, incident workflow modernization. | Teams adopting AI operations. |

## Launch Hooks

1. "Stop clicking through six dashboards to answer one incident question."
2. "Your AI assistant should read logs, deployments, incidents, and build failures before it guesses."
3. "MCP is the bridge between LLMs and real DevOps systems."
